# Hook: After Task — หลังทำงานเสร็จ

งาน **ยังไม่จบ** จนกว่าจะทำครบขั้นตอนนี้

## Checklist
1. **ทดสอบ / Verify**
   - หา test command จริงของโปรเจกต์ (`npm test`/`lint`/`pytest` ฯลฯ) แล้วรัน
   - ถ้าไม่มี test runner → ทดสอบด้วยมือ/Postman และระบุวิธี verify
   - ถ้าทดสอบไม่ผ่านหรือข้ามขั้น → **บอกตามจริง** ห้ามรายงานว่าเสร็จ
2. **QC** — ส่งให้บทบาท QC (`pj-qc`) ตรวจ contract, security/role, edge case
3. **อัปเดตความรู้ (อัตโนมัติ, ไม่สร้างซ้ำ)**
   - เจอ how-to/gotcha ใหม่ → สร้าง/แก้ไฟล์ใน [/Skills](../Skills/README.md) + อัปเดตดัชนีและ `updated:`
   - ความรู้ระดับ user/โปรเจกต์ที่ไม่ derive จากโค้ดได้ → บันทึกลง **Memory**
   - ก่อนเขียนใหม่ ตรวจของเดิมก่อน — มีหัวข้อใกล้เคียงให้แก้ไฟล์เดิม
4. **อัปเดต Documentation** — ถ้าเปลี่ยน contract/flow/schema ให้แก้ CLAUDE.md ที่เกี่ยวข้องและ `*.md` ของโดเมนนั้น
5. **เตรียม commit** — ไปต่อที่ [commit](commit.md)

## เกณฑ์ผ่าน
- ผลทดสอบถูกรายงานตามจริง
- ความรู้/เอกสารที่เปลี่ยนถูกอัปเดตแล้ว ไม่มีข้อมูลซ้ำซ้อน
